# Marts Reference — Chlorum Project (English)

> Document intended for LLM agents. Contains a complete description of each mart,
> including purpose, columns, units, source tags, and applied business rules.
> Use this file to understand what to query and how to interpret each column
> before generating queries or explanations.

All marts live in schema `dbt._producao` (prefix generated by the
`generate_schema_name` macro). All series are time series keyed by `dt`
(`timestamptz`). Grafana applies visual reductions (last, sum, avg) — the mart
always exposes the full series.

## 1. `producao__mb_45`

**Group:** MB_45 — Bipolar Membrane Electrolyzer 45
**Purpose:** Real-time monitoring of membrane operation: electric current, average voltage per
electrode, operational availability, and Cl2 production efficiency.

**Granularity:** 1 record per minute of operation.

**Business rules applied before the mart:**
- Voltage is automatically zeroed when the plant is off (`Planta Membrana Ligada = 0`).
- Total voltage divided by 92 electrodes to obtain average cell voltage.
- Availability calculated as `horas_on / 24 × 100`, capped at 100%.
- Efficiency calculated via electrochemical formula: `sensor_value / (filtered_current × 1.4923 × n_cell × hours_on × 100)`.
- `producao_cl2` calculated once per day and repeated each minute via date JOIN.

**Source tags:**

| Tag | Instrument | Role |
|---|---|---|
| `II113RC001\U` | Membrane ammeter | Plant current (kA) |
| `EI113RC001\U` | General voltmeter | Total membrane voltage (V) |
| `Planta Membrana Ligada` | PLC status | On/off gate for voltage |
| `WQI13107_100\V` | Caustic soda totalizer | Efficiency calculation base |

**Columns:**

| Column | Type | Unit | Description |
|---|---|---|---|
| `dt` | timestamptz | — | Record timestamp. Main series axis. |
| `corrente` | numeric | kA | Current read from tag `II113RC001\U`. Raw MAX per minute. Not zeroed when the plant is off — use `tensao_media = 0` as a proxy for on/off state. |
| `tensao_media` | numeric | V | Average voltage per electrode. Calculated as `soma_tensao / 92`. Equals **0** when `Planta Membrana Ligada = 0`, indicating the plant was off during that minute. |
| `horas_on` | numeric | hours | Hours during which current (`II113RC001\U`) exceeded 2.8 A on the day corresponding to `dt`. Daily value repeated each minute of the day. |
| `disponibilidade` | numeric | % | Percentage of time in operation during the day: `horas_on / 24 × 100`. Capped at 100%. Daily value repeated per minute. |
| `eficiencia` | numeric | % | Faradaic efficiency of the day's electrochemical conversion. Calculated from the delta of the caustic soda totalizer (`WQI13107_100\V`), filtered current, number of active cells, and operating hours. Daily value repeated per minute. |
| `producao_cl2` | numeric | ton | Cl2 production for the day in tonnes. Estimated by: `1.4923 × avg_current × efficiency × (cell_seconds / 3600) × 0.8864`. Daily value repeated per minute. |
| `plan_diario` | numeric | ton | Daily production target. Fixed value of **42.7 tonnes/day**, defined by process engineering. Repeated each minute. |

**Reference Grafana query:**
```sql
SELECT dt, corrente, tensao_media, disponibilidade, eficiencia, producao_cl2, plan_diario
FROM dbt._producaoproducao__mb_45
WHERE $__timeFilter(dt)
ORDER BY dt
```

## 2. `producao__solenis`

**Group:** Solenis — By-product sales
**Purpose:** Minute-by-minute monitoring of chlorine, steam, and caustic soda flow sold to
Solenis. Includes binary "no-sale" flags used by panels to calculate downtime minutes.

**Granularity:** 1 record per minute.

**Business rules applied before the mart:**
- Negative values zeroed for all flow columns.
- `sem_cloro`: triggered when `venda_cloro ≤ 0.2` (business rule — flows this low are treated
  as no effective sale, not merely zero).
- `sem_vapor` and `sem_soda`: triggered when sale is `≤ 0`.
- The Grafana panel performs `COUNT(*) WHERE sem_X = 1` to compute total no-sale minutes.

**Source tags:**

| Tag | Instrument | Role |
|---|---|---|
| `FI01507\U` | Flow meter | Chlorine flow sold |
| `FIC076012\PV_IN` | Flow controller | Steam flow sold |
| `Soda_solenis` | Soda tag | Caustic soda flow sold |

**Columns:**

| Column | Type | Unit | Description |
|---|---|---|---|
| `dt` | timestamptz | — | Record timestamp. |
| `venda_cloro` | numeric | — | Chlorine flow sold (`FI01507\U`). Negatives zeroed. |
| `venda_vapor` | numeric | — | Steam flow sold (`FIC076012\PV_IN`). Negatives zeroed. |
| `venda_soda` | numeric | — | Caustic soda flow sold (`Soda_solenis`). Negatives zeroed. |
| `sem_cloro` | integer | 0 or 1 | No chlorine sale flag. Equals **1** when `venda_cloro ≤ 0.2`. The 0.2 threshold is a documented business rule — below this, no effective sale is considered. |
| `sem_vapor` | integer | 0 or 1 | No steam sale flag. Equals **1** when `venda_vapor ≤ 0`. |
| `sem_soda` | integer | 0 or 1 | No caustic soda sale flag. Equals **1** when `venda_soda ≤ 0`. |

**Reference Grafana query:**
```sql
SELECT dt, venda_cloro, venda_vapor, venda_soda, sem_cloro, sem_vapor, sem_soda
FROM dbt._producaoproducao__solenis
WHERE $__timeFilter(dt)
ORDER BY dt
```

## 3. `producao__hcl`

**Group:** HCl — Hydrochloric Acid Production
**Purpose:** Time series of incremental HCl production, its concentration, and the fraction of
installed capacity in use at each minute.

**Granularity:** 1 record per minute.

**Business rules applied before the mart:**
- `producao_hcl` is an **incremental delta**: the `Producao_HCL` sensor is a cumulative
  totalizer. The calc layer applies `LAG` and `GREATEST(..., 0)` to avoid negative values
  on counter resets.
- `capacidade_atual` is normalized by the plant's maximum capacity (495 flow units = 100%).
  Zeroed when raw flow is ≤ 0.
- `concentracao` is the per-minute average without additional unit transformation.

**Source tags:**

| Tag | Instrument | Role |
|---|---|---|
| `Producao_HCL` | Cumulative totalizer | Production delta base |
| `Concentracao_HCL` | Analyzer | Percentage concentration |
| `FIC051025C\PV_IN` | Flow controller | Inlet flow for capacity calculation |

**Columns:**

| Column | Type | Unit | Description |
|---|---|---|---|
| `dt` | timestamptz | — | Record timestamp. |
| `producao_hcl` | numeric | ton | Incremental HCl production in the interval up to this `dt`. Difference between the current and previous value of the `Producao_HCL` totalizer. Never negative. |
| `concentracao` | numeric | % | HCl concentration at the minute (`Concentracao_HCL`). Per-minute average, no unit transformation. |
| `capacidade_atual` | numeric | ratio (0–1) | Fraction of installed capacity in use. Calculated as `FIC051025C\PV_IN / 495`. **0** when raw flow is ≤ 0. **1.0** represents 100% of capacity. |

**Reference Grafana query:**
```sql
SELECT dt, producao_hcl, concentracao, capacidade_atual
FROM dbt._producaoproducao__hcl
WHERE $__timeFilter(dt)
ORDER BY dt
```

## 4. `producao__hipo`

**Group:** Hipo — Sodium Hypochlorite Production
**Purpose:** Time series of sodium hypochlorite (NaClO) production process metrics: production,
caustic soda consumption, capacity, and water flow rate.

**Granularity:** 1 record per minute.

**Business rules applied before the mart:**
- `naclo`: flow reading converted from internal unit to tonnes (÷ 60,000). Negatives = 0.
- `consumo_naoh`: amplified by stoichiometric factor (× 357). Negatives = 0.
- `capacidade`: calculated as `naoh_raw × (350 / 15)` — represents the theoretical maximum
  capacity given the current NaOH flow.
- `vazao_agua`: negatives zeroed.

**Source tags:**

| Tag | Instrument | Role |
|---|---|---|
| `FI_NaCl0\U` | Flow meter | NaClO production |
| `FIC12702\PV_IN` | Flow controller | NaOH flow for hypochlorite production |
| `FIC12703\PV_IN` | Flow controller | Dilution water flow |

**Columns:**

| Column | Type | Unit | Description |
|---|---|---|---|
| `dt` | timestamptz | — | Record timestamp. |
| `naclo` | numeric | ton | Sodium hypochlorite (NaClO) production at the minute. Derived from `FI_NaCl0\U / 60,000`. Negatives = 0. |
| `consumo_naoh` | numeric | — | Caustic soda (NaOH) consumption, calculated as `FIC12702\PV_IN × 357`. Represents the quantity of NaOH consumed in the hypochlorite process. Negatives = 0. |
| `capacidade` | numeric | — | Hypochlorite production capacity calculated as `FIC12702\PV_IN × (350 / 15)`. Represents the theoretical production ceiling given the current NaOH flow. |
| `vazao_agua` | numeric | — | Dilution water flow rate (`FIC12703\PV_IN`). Negatives zeroed. |

**Reference Grafana query:**
```sql
SELECT dt, naclo, consumo_naoh, capacidade, vazao_agua
FROM dbt._producaoproducao__hipo
WHERE $__timeFilter(dt)
ORDER BY dt
```

## 5. `producao__liquefacao`

**Group:** Liquefação — Liquid Cl2 Compression
**Purpose:** Time series of the Cl2 compression rate in each storage tank, plus the status of
CPC compressors and the capacity of Sabroe refrigeration compressors. Used to monitor tank
filling rates and the health of the liquefaction system.

**Granularity:** 1 record per minute (already pivoted by tank — one row per `dt` with one column per tank).

**Business rules applied before the mart:**
- Weights outside `[-900, 130,000]` g are discarded (noise / sensor error).
- `taxa_compressao` calculated via **linear regression** (`regr_slope`) over the last 10 weight
  records of each tank, multiplied by 3,600 to convert from g/s to g/h ≈ kg/h.
- Rates ≥ 4,500 kg/h are discarded (outliers).
- The calc layer pivots the long series (one row per tank) to wide series (one column per tank).

**Source tags:**

| Tag | Instrument | Role |
|---|---|---|
| `Peso Tanque de Cloro 01` to `06` (LIKE `Peso%Tanque%`) | Load cells | Raw Cl2 weight per tank |
| `YLL CCL-COM-CPC - A` | PLC | CPC-A compressor status |
| `YLL CCL-COM-CPC- B` | PLC | CPC-B compressor status |
| `YLL CCL-COM-CPC - C` | PLC | CPC-C compressor status |
| `YLL CCL-COM-CPC- 3` | PLC | CPC-3 compressor status |
| `Capacidade - Sabroe 01` | PLC | Sabroe 01 percent capacity |
| `Capacidade - Sabroe 02` | PLC | Sabroe 02 percent capacity |

**Columns:**

| Column | Type | Unit | Description |
|---|---|---|---|
| `dt` | timestamptz | — | Record timestamp. |
| `cpc_tanque_01` | numeric | kg/h | Compression (filling) rate of Chlorine Tank 01. Derived from linear regression over the last 10 weight readings. Null or absent when fewer than 10 valid readings are available. |
| `cpc_tanque_02` | numeric | kg/h | Compression rate of Chlorine Tank 02. Same logic as Tank 01. |
| `cpc_tanque_03` | numeric | kg/h | Compression rate of Chlorine Tank 03. |
| `cpc_tanque_04` | numeric | kg/h | Compression rate of Chlorine Tank 04. |
| `cpc_tanque_05` | numeric | kg/h | Compression rate of Chlorine Tank 05. |
| `cpc_tanque_06` | numeric | kg/h | Compression rate of Chlorine Tank 06. |
| `status_cpc_a` | numeric | 0 or 1 | CPC-A compressor operating status. **1** = on/running, **0** = off. |
| `status_cpc_b` | numeric | 0 or 1 | CPC-B compressor operating status. |
| `status_cpc_c` | numeric | 0 or 1 | CPC-C compressor operating status. |
| `status_cpc_3` | numeric | 0 or 1 | CPC-3 compressor operating status. |
| `cap_sabroe_01` | numeric | % | Percent capacity in use for Sabroe 01 compressor. |
| `cap_sabroe_02` | numeric | % | Percent capacity in use for Sabroe 02 compressor. |

**Reference Grafana query:**
```sql
SELECT dt, cpc_tanque_01, cpc_tanque_02, cpc_tanque_03, cpc_tanque_04,
       cpc_tanque_05, cpc_tanque_06, status_cpc_a, status_cpc_b,
       status_cpc_c, status_cpc_3, cap_sabroe_01, cap_sabroe_02
FROM dbt._producaoproducao__liquefacao
WHERE $__timeFilter(dt)
ORDER BY dt
```

## 6. `producao__estoque_cloro`

**Group:** Estoque Cloro — Liquid Cl2 Inventory
**Purpose:** Time series of total liquid chlorine (Cl2) inventory across all plant tanks,
expressed in tonnes.

**Granularity:** 1 record per weight sensor reading (typically per minute).

**Business rules applied before the mart:**
- Weights outside `[-900, 130,000]` g discarded per tank.
- Valid weights from all tanks summed per `dt`.
- Sum in grams converted to tonnes (÷ 1,000).

**Source tags:**

| Tag | Instrument | Role |
|---|---|---|
| `Peso%Tanque%0%` (LIKE) | Load cells | Cl2 weight per tank |

**Columns:**

| Column | Type | Unit | Description |
|---|---|---|---|
| `dt` | timestamptz | — | Record timestamp. |
| `peso_total_ton` | numeric | tonnes | Total liquid Cl2 inventory: sum of valid weights across all tanks, converted from grams to tonnes. |

**Reference Grafana query:**
```sql
SELECT dt, peso_total_ton
FROM dbt._producaoproducao__estoque_cloro
WHERE $__timeFilter(dt)
ORDER BY dt
```

**Business rules applied before the mart:**

- 95% factor applied to reflect effective usable tank capacity (`× 0.95`).

**Source tags:**

| Tag | Instrument | Role |
|---|---|---|
| `READ_TRANSMITERS\LI0510%.PV` (LIKE) | Level transmitters | HCl level per tank |

**Columns:**

| Column | Type | Unit | Description |
|---|---|---|---|
| `dt` | timestamptz | — | Record timestamp. |
| `nivel_total` | numeric | — | Total HCl level: sum of valid sensors per `dt`, multiplied by 0.95. The 95% factor is an engineering correction for the effective usable capacity of the tanks. |

**Reference Grafana query:**
```sql
SELECT dt, nivel_total
FROM dbt._producaoproducao__estoque_hcl
WHERE $__timeFilter(dt)
ORDER BY dt
```

## 10. `producao__producao_dia`

**Group:** Produção Dia — Consolidated Daily Production View
**Purpose:** The most comprehensive mart in the project. Consolidates into a single time series
all daily production metrics: caustic soda, HCl, Cl2, electrical demand, supply voltage, and
power factor. Designed for operational overview dashboards.

**Granularity:** 1 record per minute.

**Business rules applied before the mart:**
- `producao_soda` and `producao_hcl`: deltas of cumulative totalizers via `LAG + GREATEST(..., 0)`. Never negative.
- `vazao_agua`: negatives zeroed.
- `cl2_on`: binary flag derived from `FIC12718\MV ≥ 1`.
- `demanda`: converted from watts to kilowatts (÷ 1,000).
- `producao_cl2` and `plan_diario`: calculated once per day (by the `producao_cl2_base` util)
  and repeated each minute via LEFT JOIN on date.

**Source tags:**

| Tag | Instrument | Role |
|---|---|---|
| `WQI13107_100\V` | Caustic soda totalizer | Soda production delta base |
| `Producao_HCL` | HCl totalizer | HCl production delta base |
| `FIC12702\PV_IN` | Flow controller | NaOH flow rate |
| `FIC12703\PV_IN` | Flow controller | Water flow rate |
| `FIC12718\MV` | Controller | Cl2 active signal |
| `Demanda Variável` | Energy meter | Total plant electrical demand |
| `Tensão Média FF` | Electrical meter | Average supply voltage |
| `Fator de Potência` | Electrical meter | Plant power factor |
| `II113RC001\U` | Ammeter | Current (base for producao_cl2) |
| `EI113RC001\U` | Voltmeter | Voltage (base for producao_cl2) |

**Columns:**

| Column | Type | Unit | Description |
|---|---|---|---|
| `dt` | timestamptz | — | Record timestamp. Main axis. |
| `producao_soda` | numeric | — | Incremental caustic soda production in the interval up to this `dt`. Calculated as the difference between the current and previous value of the `WQI13107_100\V` totalizer. Never negative. |
| `producao_hcl` | numeric | — | Incremental HCl production in the interval up to this `dt`. Delta of the `Producao_HCL` totalizer. Never negative. |
| `vazao_naoh` | numeric | — | NaOH flow rate from tag `FIC12702\PV_IN`. No additional transformation. Can be used to monitor caustic soda consumption in the hypochlorite and electrolysis processes. |
| `vazao_agua` | numeric | — | Process water flow rate from tag `FIC12703\PV_IN`. Negatives zeroed. |
| `cl2_on` | integer | 0 or 1 | Flag indicating whether the Cl2 system is active. Equals **1** when `FIC12718\MV ≥ 1`, **0** otherwise. Used to calculate Cl2 operating time during the day. |
| `demanda` | numeric | kW | Total plant electrical demand in kilowatts. Derived from `Demanda Variável` divided by 1,000. |
| `tensao_ff` | numeric | V | Average supply voltage of the plant (`Tensão Média FF`). Per-minute average. |
| `fator_potencia` | numeric | — | Electrical installation power factor (`Fator de Potência`). Dimensionless, typically between 0 and 1. Per-minute average. |
| `producao_cl2` | numeric | ton | Estimated Cl2 production for the whole day in tonnes. Calculated via the electrochemical formula based on current, efficiency, and operating time (see MB_45 group). Daily value repeated each minute via LEFT JOIN on date. |
| `plan_diario` | numeric | ton | Daily Cl2 production target. Fixed value of **42.7 tonnes/day**, defined by process engineering. Repeated each minute. |

**Reference Grafana query:**
```sql
SELECT dt, producao_soda, producao_hcl, vazao_naoh, vazao_agua,
       cl2_on, demanda, tensao_ff, fator_potencia, producao_cl2, plan_diario
FROM dbt._producaoproducao__producao_dia
WHERE $__timeFilter(dt)
ORDER BY dt
```

## General notes for the agent

- **Always a time series:** all marts return complete series. Reductions (daily sum, last value) are the Grafana panel's responsibility, never the mart's.
- **Daily values repeated per minute:** `horas_on`, `disponibilidade`, `eficiencia`, `producao_cl2`, and `plan_diario` have daily granularity but appear repeated on each minute row via LEFT JOIN on `date_trunc('day', dt)`. When aggregating, use `MAX()` or `DISTINCT ON (date_trunc('day', dt))` to avoid double-counting.
- **95% factor in inventories:** present in `estoque_hcl`, `estoque_hipo`, and `estoque_naoh`. This represents an engineering calibration, not an error.
- **Backslash in tag names:** tags such as `II113RC001\U` contain a literal backslash. In SQL, inside a single-quoted string, `\` is literal — never double it.
- **Actual schema:** the database schema is `dbt._producao` (generated by the `generate_schema_name` macro that concatenates `target.schema` + `custom_schema_name`). The default `target.schema` is `dbt`, and `custom_schema_name` is `_producao`.
- **Incremental strategy:** all marts use `unique_key='dt'` with `WHERE dt > MAX(dt in this)`. On the first full run, they load the entire time window; on subsequent runs, only new records are appended.
