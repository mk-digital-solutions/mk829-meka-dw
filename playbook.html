Playbook Interativo: MK829 - Data Warehouse Mekatronik

Bem-vindo ao playbook interativo para o projeto MK829 - Meka-DW. Este guia cobre a configuração do ambiente, os fluxos de dados com Airbyte, os padrões de nomenclatura e as boas práticas do projeto.

📌 Visão Geral do Projeto

Este projeto visa reestruturar os dados internos de gestão da Mekatronik em um Data Warehouse (DW), focando em organização, padronização e análise de dados.

Ambiente

  • Servidor: Mekatronik
  • Sistema Operacional (Host): Windows Server 2025
  • Virtualização: Hyper-V
  • Máquina Virtual: Linux (Debian)
  • IP da VM: 192.168.0.168

Componentes

  • docker-compose.yaml: Provisiona o ambiente PostgreSQL
  • abctl: Gerencia a instalação do Airbyte via Kubernetes (K3S)

Fluxo de Dados

  1. Extração de dados das fontes (Google Sheets, FlowUp, Agilizatronik, MMGP, VOE)
  2. Ingestão via Airbyte para o schema raw_airbyte no PostgreSQL
  3. Transformação e modelagem nas camadas stg_, int_ e mart_
  4. Consumo analítico via Power BI ou outras ferramentas de BI
🏷️ Padrões de Nomenclatura
O que é snake_case? É escrever nomes usando letras minúsculas separadas por underline (_). Exemplo: nome_do_projeto, data_criacao. Bancos de dados e ferramentas como Airbyte são sensíveis a espaços e acentos — seguir esse padrão evita erros.
Schemas (Pastas do Banco)

Pense nos schemas como as pastas do banco de dados. Cada pasta guarda tabelas de uma mesma etapa do pipeline.

PrefixoExemploO que guarda
raw_raw_airbyteDados brutos, exatamente como chegam das fontes
stg_stg_faturamentoDados limpos e padronizados (staging)
int_int_projetosDados já combinados entre tabelas (intermediário)
mart_mart_gestaoDados prontos para análise e relatórios
audit_audit_logsRegistros de controle e rastreabilidade

Regras: tudo em minúsculo, sem acentos, sem espaços, use _ para separar palavras.

🔄 Fluxos de Extração com Airbyte

O Airbyte é usado para dois fluxos principais de dados.

Fluxo 1: Google Sheets → PostgreSQL

Extrai dados de planilhas no Google Sheets e carrega no PostgreSQL (schema raw_airbyte).

Configuração da Source (Google Sheets)

  1. Acesse Sources → + New Source
  2. Selecione Google Sheets
  3. Preencha:
    CampoValor
    Namegoogle_sheets_MKXXX
    AuthenticationService Account Key (JSON)
    Spreadsheet LinkURL da planilha
    Row Batch Size200
  4. Compartilhe a planilha com o e-mail da conta de serviço (permissão de Leitor ou Editor)
  5. Clique em Test and Save

Configuração da Destination (PostgreSQL)

  1. Acesse Destinations → + New Destination
  2. Selecione PostgreSQL
  3. Preencha:
    CampoValor
    Namepostgres_mkdw
    HostMeka-dw
    Port5432
    Databasepostgres
    Schemaraw_airbyte
    UsernameUsuário com permissão de escrita
    PasswordSenha do usuário
    SSL Moderequire
  4. Clique em Test and Save

Configuração do Sync

  1. Acesse Connections → + New Connection
  2. Selecione source e destination
  3. Renomeie os streams conforme os nomes definidos na seção "Fontes de Dados e Tabelas Raw"
  4. Configure: Full Refresh — Overwrite, agendamento a cada 12 horas (padrão)
  5. Clique em Set up connection
⚠️ Full Refresh — Overwrite: a cada execução, todos os dados da tabela de destino são apagados e recarregados. Alterações manuais no PostgreSQL serão perdidas no próximo sync.
Fluxo 2: PostgreSQL → Google Sheets

Exporta dados do PostgreSQL para planilha de gestão no Google Sheets.

Configuração da Source (PostgreSQL)

  1. Acesse Sources → + New Source
  2. Selecione PostgreSQL
  3. Preencha:
    CampoValor
    Namepostgres_mkdw
    HostMeka-dw
    Port5432
    Databasepostgres
    SchemaSchema das tabelas exportadas
    UsernameUsuário somente leitura
    PasswordSenha
    SSL Moderequire
  4. Clique em Test and Save
💡 Use sempre um usuário com permissão somente leitura na source para evitar qualquer risco de alteração nos dados de origem.

Configuração da Destination (Google Sheets)

  1. Acesse Destinations → + New Destination
  2. Selecione Google Sheets
  3. Preencha:
    CampoValor
    NamePLANILHA_GESTÃO
    AuthenticationService Account Key (JSON)
    Spreadsheet LinkURL da planilha de gestão
  4. Compartilhe a planilha com permissão Editor para a conta de serviço
  5. Clique em Test and Save
💡 Cada stream (tabela) do PostgreSQL será carregado em uma aba separada da planilha. Nomeie as abas de forma consistente com os nomes das tabelas de origem.

Configuração do Sync

  1. Acesse Connections → + New Connection
  2. Selecione source e destination
  3. Configure: Full Refresh — Overwrite, a cada 12 horas
  4. Clique em Set up connection

💡 Boas Práticas
  • Siga sempre os padrões de nomenclatura definidos neste playbook
  • Nomeie conexões e streams de forma descritiva e consistente com os nomes das tabelas
  • Monitore execuções regularmente em Connections → Sync History
  • Gerencie credenciais com segurança — nunca salve senhas em texto plano ou no repositório
  • Evite editar planilhas de gestão manualmente — alterações são perdidas no próximo sync
  • Escalone horários de sync para não sobrecarregar o banco ao mesmo tempo
